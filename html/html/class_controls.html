Descrição Detalhada

The Controls class provides support to user control.

This class allows to programm the interface for dealing with play/pause/prev/next/volume/elapsed time commands

Construtores & Destrutores

Controls::Controls ( QWidget *  parent = 0)
explicit

Class constructor.

Parâmetros
parentis the pointer to parent widget (it is supposed to be the place where this widget has to be drawn)

Métodos

void Controls::onDurationChanged ( qint64  value)
slot

onDurationChanged slot is called when a new media arives from the playlist

When a new media starts to play, mainwindow tells this widget the total amount of time associated to this media.

Parâmetros
valuestores the amount of time for the current media
void Controls::onElapsedChanged ( qint64  value)
slot

onElapsedChanged is signal activated periodically by mainwindow widget

every a new part of the song is played, the stream position changes. This position is comunicated to controls widget using some specific signal.

We need two overloaded function, since the scrollbar widget only deals with integer values and the signal provided by media player offers a 64 bits integer :p

Parâmetros
valuereceives the elapsed time
void Controls::onPlayerStateChanged ( bool  state)
slot

onPlayerStateChanged deals with the state of play/pause button

The developer may want to state different icons for each state

Parâmetros
statestores play/pause state
  • true: Player is on playing state
  • false: Player is on paused state
void Controls::onSliderReleased ( )
protectedslot

onSliderReleased is called when the user releases the duration slider

When this slider is released, the widget tells mainwindow that user selected a new part of the song to play.
